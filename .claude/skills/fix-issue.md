# 問題修正スキル

## 手順

1. 問題のレポートや指摘事項を確認する
2. 該当するソースコードを特定・修正する
3. テストコードを必ず追加・修正する
4. コミット前チェックを実施（以下すべて）

## 横展開チェック

- 修正した問題と同じパターンが他ファイルにないか検索
- テストコード内で旧文言を検索し全て新文言に修正
- 共通処理の変更は関連する全ファイルに横展開

## セキュリティチェック

- ユーザー入力のサニタイズ（HTML/SQL/CSS）
- パラメータ化クエリの使用
- 入力バリデーションの設定
- 外部ファイルのサイズ上限・型チェック
- 機密情報のハードコード禁止

## パフォーマンスチェック

- ループ内DB問い合わせ禁止（一括取得→メモリ上フィルタ）
- 描画オブジェクトのキャッシュ
- 不要な再描画の抑制
- 独立した非同期操作の並列化
