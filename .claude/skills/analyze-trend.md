# トレンド分析スキル

ブログ記事の作成・編集前に実行し、最新トレンドを分析して蓄積する。

## 実行手順

### Step 1. トレンド記事の取得

以下のソースからトレンド上位10記事を取得する。

#### Qiita トレンド

1. `https://qiita.com/` にアクセスし、トレンド一覧を取得
2. 上位10記事のタイトル・URL・いいね数・タグを記録
3. 各記事にアクセスし、以下を分析:
   - タイトル構造（文字数、感情語、キーワード配置）
   - 冒頭の導入パターン（問いかけ/断言/データ提示）
   - 記事構成（段階的/比較表/ハンズオン/ストーリー）
   - タグ戦略（季節性、具体性、組み合わせ）
   - 読者エンゲージメントを高めている要素

#### Zenn トレンド（可能な場合）

1. `https://zenn.dev/` にアクセスし、トレンド一覧を取得
2. 上位5記事について同様の分析を実施

### Step 2. 分析レポートの作成

以下のテンプレートで `docs/TrendAnalysis/YYYY-MM-DD_<source>.md` に保存する:

```markdown
---
date: YYYY-MM-DD
source: qiita | zenn
articles_analyzed: 10
---

# トレンド分析レポート — YYYY-MM-DD（Qiita）

## 分析サマリー

| 順位 | タイトル | いいね | タグ |
|------|---------|--------|------|
| 1 | ... | ... | ... |
| ... | ... | ... | ... |

## 成功パターン分析

- 平均文字数: X文字
- 多用されるパターン: （感情語+サブタイトル / 数字+具体的成果 / 問いかけ型）
- 今週特有のキーワード: ...

### 構成傾向
- 主流の記事構成: （段階的 / 比較表 / ハンズオン / ストーリー）
- 冒頭パターン: （問いかけ / 断言 / データ提示 / 個人体験）
```

### Step 3. TREND_INDEX.md の更新

`docs/TrendAnalysis/TREND_INDEX.md` に以下を追記:
- 新しい分析レポートへのリンク
- 「累積パターン」セクションを更新（全レポートから抽出した横断的傾向）

### Step 4. 累積パターンの抽出

蓄積された全レポートを読み込み、以下の累積パターンを TREND_INDEX.md に反映:

1. **常に有効なパターン**（複数回のレポートで繰り返し確認されたもの）
2. **時期依存パターン**（特定の季節・イベントで有効なもの）
3. **新興パターン**（直近で初めて観測されたもの）
4. **衰退パターン**（以前は有効だったが効果が薄れているもの）

## 注意事項

- 分析は客観的に行い、データに基づく
- 自分の記事との差分を意識し、具体的な改善提案を含める
- 過去の分析と比較し、トレンドの変化を記録する
- 分析レポートはリポジトリにコミットし、履歴として残す
