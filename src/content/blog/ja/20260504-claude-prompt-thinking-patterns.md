## 連載第 7 回です

[前回](/HomePage/ja/blog/20260503-claude-code-features/)で Claude Code 固有機能をお伝えしました。今回は少し視点を変えて、**コードを書かない用途での Claude** です。

Claude は「コード生成 AI」として語られがちですが、実際には **情報収集・学習・意思決定** といった思考系タスクでも抜群に強い。私は日常的に Claude を「一緒に考えてくれる相棒」として使っています。

## B. 情報収集・学習系

### パターン 1. 技術調査

```
{{技術名}} について、次の観点で調査してください:

- 概要（3 行で）
- 似た技術との比較（最低 3 つ）
- 採用するメリット/デメリット
- 代表的な採用企業・事例
```

### パターン 2. 公式ドキュメント読解

```
次の公式ドキュメントを読んで、以下を教えてください:
{{URL または貼り付け}}

- 主要な 5 つの概念
- 初学者が最初に躓くポイント
- 30 分で実装できるミニプロジェクト例
```

### パターン 3. 学習計画の作成

```
{{目標}} を達成するため、{{期間}} で学習計画を立ててください。

- 私の現状: {{経験と知識レベル}}
- 1 週間ごとのマイルストーン
- 各週で読むべき資料・作るべき成果物
```

### パターン 4. 比較検討

```
{{A}} と {{B}} と {{C}} を、次の観点で比較してください:
- 学習コスト / 運用コスト / エコシステム / 採用事例

出力: マトリクス表 + 各軸で最もバランスの良い選択と理由
```

### パターン 5. 既存概念の再整理

```
私が理解している {{概念}} は次の通りです:
{{自分の理解を箇条書き}}

これを読んで:
- 正しい理解
- 誤解している点
- 抜けている重要概念
を指摘してください。
```

**これが最強**。自分の理解を言語化して Claude に添削してもらうと、ブレや抜けが可視化されます。

### パターン 6. ケーススタディ収集

```
{{課題}} を解決した実例を 5 つ集めてください。

各事例で:
- 会社名（公開情報の範囲で）
- 採用技術
- 判断の決め手
- 結果（数値）
```

### パターン 7. 反対意見を集める

```
{{自分の主張}} に対する反対意見を、最低 5 つ挙げてください。

- それぞれの反対意見の論拠
- どの反対意見が最も強いか
- 自分の主張を補強するための追加情報
```

**自分の盲点を炙り出す** 用途で毎週使っています。

## C. 思考・意思決定系

### パターン 8. 論点整理

```
以下の議論を読んで、論点を整理してください:
{{議事録や対話ログ}}

出力:
- 論点を最大 5 つまで抽出
- 各論点の「賛成意見」と「反対意見」
- 未決事項と次に判断すべき人
```

### パターン 9. 意思決定フレームワーク

```
{{意思決定}} について、以下のフレームで検討してください:

- Pros（最低 5 つ）
- Cons（最低 5 つ）
- やらなかった場合のリスク
- リバーシブルか、イリバーシブルか
- 1 年後振り返ったらどう見えるか
```

**「1 年後振り返ったら」** の問いが、短期思考を抜け出すのに効きます。

### パターン 10. ディベート

```
私は {{立場 A}} の立場です。あなたは {{立場 B}} の立場で反論してください。
3 ラウンド議論しましょう。最後に、議論の結論として
どちらの立場がどの観点で優勢だったかを総括してください。
```

### パターン 11. 要約

```
次の長文を要約してください:
{{長文}}

レベル別に 3 段階:
- 1 行要約（140 字以内）
- 3 行要約
- 20 行要約（重要論点すべて）
```

### パターン 12. 思考フレームの応用

```
{{課題}} を、次のフレームで分析してください:
- MECE で分解
- 5W1H で整理
- Before/After で変化を可視化
```

### パターン 13. メタ認知

```
私は最近 {{悩み}} で迷っています。
- 私がそもそも解くべき問題は何か
- この悩みの本質は何か
- 先送りしている決断は何か
を整理してください。
```

**自分だけで考えると堂々巡りするとき** に使う。Claude は「そもそも問題設定が間違っていませんか？」と遠慮なく指摘してくれます。

## パターン活用のコツ

| コツ | 理由 |
|---|---|
| 出力形式を明示 | 思考系は雑に書くと要領得ない回答になる |
| 反対意見を必ず入れる | Claude は「お世辞」を言いがち。意識的に反論させる |
| 複数パターンを連鎖 | パターン 5（再整理）→ 7（反対意見）→ 9（意思決定）が最強ルート |

## 私の体感 — Claude は「考える仲間」

以前、技術選定で迷ったとき、同僚と相談してもなかなか結論が出ないことがありました。今は **まず Claude とディベートする** ようにしています。

- 自分の主張を Claude に反論させる → 盲点が見える
- 複数の候補を Claude に比較させる → 客観的な評価軸が揃う
- 決めた結論を「1 年後振り返り」視点で検証 → 短期バイアスを外せる

結果、同僚との相談もより **論点が絞れた状態** でできるようになりました。Claude は代替ではなく **思考の準備運動** として最高の相棒です。
