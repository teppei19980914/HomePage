## はじめまして

須山 哲平です。この度、個人ホームページを開設しました。

普段は正社員として SES エンジニアをやりながら、副業でフルスタック開発に従事しています。

## なぜホームページを作ったのか

私は、IT業界で 5 年間、PG → SE → PL とキャリアを積んできました。業務の非効率を自ら見つけて改善を提案・実行し、退行テスト業務 90% 削減、残業時間 60% 削減、月 40 人時の完全 RPA 化など、定量的な成果を出してきました。

しかし、これらの実績は社内でしか知られていません。もっと多くの人に自分の実績や考え方を知ってもらい、「この人と一緒に仕事がしたい」と思ってもらえる場所が欲しい。
それがこのホームページを作成しようと考えたきっかけです。

これまで、私の情報は Qiita、Wantedly、GitHub、個人開発アプリなど、さまざまな場所に**散らばっていました**。どれか1つを見ただけでは、私の全体像は伝わりません。  
つまり、このホームページの意義としては、私の全体像を把握してもらうために情報を一元管理するためのものです。

## このホームページで成し遂げたいこと

### 1. 案件発注者・採用担当者に自分を知ってもらう

スキル・資格・キャリアといった客観的な情報だけでなく、私の哲学や夢、座右の銘まで掲載しています。技術力だけでなく、人物像も含めて判断していただきたいと考えています。

### 2. 個人開発プロダクトを発信する

現在、2つのアプリを個人で開発しています。

- **ユメハシ（YumeHashi）** — 夢を目標に分解し、日々の行動に変えるアプリ。Flutter / Dart で開発。
- **Defrago** — 脳内メモリをデフラグ化し、エッセンシャル思考をもとにタスクを整理するアプリ。Python / FastAPI で開発中。

どちらも AI 駆動開発（Claude Code）で開発しており、ユメハシは未経験技術で 3 週間、Defrago は一次リリースを 2 週間で達成しました。
（Defrago はアプリ改修のため、開発中としています。）

### 3. 技術ブログで知見を発信する

[Qiita](https://qiita.com/teppei19980914) では 211 件以上の技術記事を投稿してきました。このブログでは Qiita とは異なる切り口で、以下のテーマを扱っていきます。

- 個人開発の振り返りと学び
- AI 駆動開発のリアルな体験記
- キャリアについての考え
- プロジェクトマネジメントの知見
- 読書や日々の気づき

## 「高機能をシンプルに」— エッセンシャル思考

このホームページも、そして今までの開発も同じ思想で設計しました。  

- 導線は必要最小限（ホーム / プロフィール / プロダクト / プロジェクト / ブログ / お問い合わせ）
- スクロールが長くなりがちな プロフィール は目次（TOC）でセクションジャンプできる設計
- テキストは多言語対応を見据えて `src/i18n/ja.ts` に一元管理し、ハードコーディング禁止
- クライアント JS は最小限（Astro のゼロ JS デフォルトを活用）

「情報を詰め込む」のではなく、「ユーザーが迷わず、最速で必要な情報にたどり着ける」ことを最優先にしています。

## このサイトの技術構成

エンジニアとして、このサイト自体も技術的なこだわりを持って構築しました。

- **Astro v6** — JS ゼロデフォルトの静的サイト生成。パフォーマンス最優先
- **GitHub Pages** — コスト 0 円でホスティング
- **Cloudflare Web Analytics** — Cookie 不要のアクセス解析
- **Formspree** — Cookie 不要のお問い合わせフォーム
- **SEO 最適化** — sitemap / OGP / JSON-LD / RSS / Google Search Console
- **AI 駆動開発** — Claude Code でサイト全体を構築
