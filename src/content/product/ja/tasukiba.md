> **知見を残す。判断をつなぐ。プロジェクトを強くする。**
>
> プロジェクトの企画から振り返りまでを、ナレッジを中心に一気通貫で支援する業務管理プラットフォームです。

> 本記事は約 **5 分** で読めます。

## コンセプト — 運営するほど、次のプロジェクトがうまくいく

プロジェクトの現場では、毎回のように同じ失敗が繰り返され、貴重な判断や学びが個人の頭の中やバラバラのドキュメントに埋もれていきます。担当者が離れると、せっかくの知見もそのまま失われてしまう。

「**あのときこの知見があれば、見積もりはもっと精度高く立てられた**」
「**前任者が残してくれていたら、同じトラブルで半日を溶かさずに済んだ**」

こうした現場の悔しさを、仕組みで減らしたい。それが **たすきば** の出発点です。

サービス名の「たすきば (襷場)」は、ランナーが次のランナーへ襷をつなぐ場所から来ています。プロジェクトという一区間を走り抜けた担当者が、**次の担当者へ知見の襷を確実に渡せる場所** を目指しています。

## 解決する課題

| 現場の課題 | たすきばでの解決 |
|---|---|
| 見積もりの根拠が残らず、毎回ゼロから試算 | 見積工数・根拠・関連ナレッジを履歴として蓄積 |
| リスクと対応策が散在し、再発防止が効かない | リスク・課題管理にナレッジを紐付けて再発を抑制 |
| 振り返りが形骸化し、次案件に活かされない | 振り返り → ナレッジ化 → 次案件参照 の循環を作る |
| 属人化で回り、担当者が抜けると崩壊する | プロジェクト・タスク・ナレッジを構造化して継続利用 |
| 品質・コスト・納期のバランスが見えない | 計画と実績を一気通貫で可視化 |

## 主な機能

プロジェクト運営に必要な機能を、**ナレッジを中央に据えて**一つの基盤でつなぎました。

| 機能 | できること |
|---|---|
| **プロジェクト管理** | 企画から振り返りまでの状態遷移を管理 |
| **見積もり管理** | 過去の実績・ナレッジを参照した精度の高い見積作成 |
| **WBS / タスク管理** | 階層化されたタスクの担当割り当てと進捗更新 |
| **ガントチャート** | スケジュール・進捗・遅延・マイルストーンの時系列可視化 |
| **リスク・課題管理** | 起票から対応策の記録まで、CSV エクスポートにも対応 |
| **ナレッジ管理** | 全文検索付きの知見ベース。プロジェクト横断で再利用可能 |
| **振り返り** | 完了後の総括 (KPT 形式) を自動的にナレッジへ昇格 |
| **マイタスク** | 個人単位の担当タスク一覧と進捗更新ショートカット |

### 「ナレッジが中心」ということの意味

他のプロジェクト管理ツールの多くは、タスク管理と知見管理を別モジュールに置いています。たすきばは **ナレッジを中心** に据え、見積もり・タスク・リスク・振り返りがすべてナレッジと双方向に紐付く設計です。

日々の業務をこなすだけで、自然と「次のプロジェクトで誰かが助かる知見」が育っていきます。

## こだわり — このサービスの強み

### 1. ナレッジを中核にした「提案型」設計

プロジェクトの業務ドメイン・工程・技術スタックをキーに、**過去プロジェクトから関連する知見・リスク・振り返りを自動提案**します。「どんな知見があるか分からないから探せない」を解消します。

### 2. セキュリティに正面から向き合う

業務データを預かるサービスとして、セキュリティは最重要領域として扱っています。

- 多要素認証 (MFA) 対応。管理者ロールは MFA 必須
- ロールベースの権限管理で、他プロジェクトへの不正アクセスを防止
- 全データ変更・認証イベント・権限変更履歴の自動記録
- パスワードポリシー・アカウントロック・未使用アカウント自動削除
- 開発工程で 6 層のセキュリティスキャンを自動実行
- 四半期ごとの脅威モデリングを運用として制度化

### 3. 品質を数字で担保する

「動くサービス」ではなく「動き続けるサービス」を目指し、品質を数値で管理しています。

- ユニットテスト **646 件** 全 pass を継続維持
- コードカバレッジ **80%** を CI で強制（未達ならビルド失敗）
- E2E テストと視覚回帰テストを 10 テーマ分の画面で自動検証

### 4. AI 駆動開発を前提に設計

本サービスは **AI 支援を前提に設計されたリポジトリ** です。開発の再現性と品質を担保するため、コミット前の自動チェック、危険コードの検知、ドキュメント更新の連動などを仕組み化しています。少人数でも高い品質を維持できる体制が、このサービスそのものに反映されています。

### 5. ドキュメントを「コードと同格の資産」として扱う

役割別 (初見開発者 / 開発者 / 運用管理者) のドキュメント構造、30 分で開発環境構築から PR 作成まで到達できる初見向けガイド、過去の意思決定の履歴資料など、**「AI がなくても人間が引き継げる状態」** を目標に整備しています。サービスの将来を、特定個人の頭の中に依存させません。

## 技術スタック

| レイヤー | 採用技術 |
|---|---|
| フロントエンド | Next.js 16 (App Router) / React 19 / TypeScript |
| UI | shadcn/ui / Tailwind CSS |
| バックエンド | Next.js API Routes / Server Actions |
| データベース | PostgreSQL 16（全文検索は pg_trgm で実装し、外部検索エンジン不要） |
| ORM | Prisma 7 |
| 認証 | NextAuth.js (Auth.js) 5 + TOTP MFA |
| テスト | Vitest / Playwright |
| デプロイ | Vercel + Supabase |

**ゼロランタイムコスト** の運用を目指し、プレリリースまでは Vercel Hobby + Supabase Free の無料枠で稼働しています。

## 最後に

たすきばは「派手な新機能」で驚かせるサービスではありません。**プロジェクトを繰り返すほど、組織の知見が確実に蓄積されていく**。その地味で、しかし本質的な仕組みを、正面から作っています。

プロジェクト現場の悔しさを仕組みで減らし、次のランナーに襷を確実に渡せる場所。それがこのサービスの役目です。
