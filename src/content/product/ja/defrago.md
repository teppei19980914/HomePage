## 脳の中を、デフラグする。

「あれもやらなきゃ」「これも忘れちゃいけない」——頭の中がタスクで散らばっていませんか？

脳のワーキングメモリは極めて有限です。未処理のタスクが断片化してメモリを占有し続ける限り、本来の創造的な処理能力は発揮できません。

**Defrago** は、PC のディスクデフラグのように、頭の中に散らばった気がかりやタスクを整理して最適配置するツールです。エッセンシャル思考で機能を徹底的に絞り込み、シンプルかつ高速にタスク管理できることを目指しています。

> **Defrago** = Defragment + Go  
> 散らばった思考を整理し、すぐに行動に移す。

## コンセプト

**高機能をシンプルに（エッセンシャル思考）**

必要な機能だけを磨き上げ、不要な機能は徹底的に削ぎ落とす。ユーザーが迷わず、考えさせず、最速でタスクに落とし込める設計を追求しました。

「より多く」ではなく「より少なく、しかし本質的に」。この思想が Defrago のすべての設計判断の根幹にあります。

## 3つの設計原則

| 原則 | 説明 |
|------|------|
| **Zero-Friction Capture** | 書き出す行為のハードルを限りなくゼロに。思いついた瞬間に外部化する |
| **One Screen, One Decision** | 各画面で求める判断は常に1つだけ。迷わない、考えさせない |
| **Think Less, Do More** | アプリが判断フローを導く。管理に頭を使わず、行動に集中する |

## 主な機能

### 4つのフェーズ

```
収集 → 明確化 → 実行 → 見直し → (循環)
```

| フェーズ | できること |
|---------|-----------|
| **収集** | 気になることをすべて Inbox に書き出す。ゼロフリクションで脳を空にする |
| **明確化** | ウィザードが Yes/No で導く。タスク・プロジェクト・即実行などに自動分類 |
| **実行** | 今やるべきタスクをフィルタリング。ステータス変更はワンタップ |
| **見直し** | 完了タスクの振り返りとプロジェクトの再計画。サイクルを回し続ける |

### 重要度 × 緊急度マトリクス

| 象限 | 重要度 | 緊急度 | アクション |
|------|--------|--------|-----------|
| **Q1** 必須の領域 | 高 | 高 | 今すぐ実行 |
| **Q2** 効果性の領域 | 高 | 低 | 計画を立てる |
| **Q3** 錯覚の領域 | 低 | 高 | 委任を検討 |
| **Q4** 浪費の領域 | 低 | 低 | 後回し/削除 |

## アーキテクチャ

```
src/defrago/
├── models.py              # データモデル（StrEnum + dataclass）
├── repository_protocol.py # リポジトリProtocol
├── logic/                 # ビジネスロジック（Web非依存）
│   ├── collection.py      # 収集
│   ├── clarification.py   # 明確化
│   ├── execution.py       # 実行
│   └── review.py          # 見直し
└── web/                   # FastAPI Webアプリケーション
    ├── app.py             # アプリファクトリ
    ├── routers/           # 各フェーズのルーター
    ├── templates/         # Jinja2 + HTMXパーシャル
    └── static/            # CSS + HTMX
```

### 設計方針

- **ロジック層分離**: リポジトリ Protocol を介し、ビジネスロジックを Web 層から完全分離
- **3層アーキテクチャ**: Model → Logic → Web
- **HTMX**: ページ遷移なしのインタラクティブ UI。SPA の体験をサーバーサイドで実現
- **SQLite 永続化**: 軽量でゼロ設定
