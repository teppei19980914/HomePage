## 脳の中を、デフラグする。

「あれもやらなきゃ」「これも忘れちゃいけない」——頭の中がタスクで散らばっていませんか？

脳のワーキングメモリは極めて有限です。未処理のタスクが断片化してメモリを占有し続ける限り、**人間がパフォーマンスを発揮しなくてはいけないタスクに集中できません**。

**Defrago** は、PC のディスクデフラグのように、頭の中に散らばった気がかりやタスクをデフラグメンテーション（断片化を解放）し、脳のリソースを本来集中すべきタスクに向けるためのツールです。

> **Defrago** = Defragment + Go
> 散らばった思考を整理し、すぐに行動に移す。

## コンセプト

**ゴールに最短で到達できるシンプルさ**

不要な機能は徹底的に削ぎ落とし、ユーザーが迷わず、考えさせず、最速でタスクに集中できる設計を追求しました。「より多く」ではなく「より少なく、しかしゴールへの最短経路を」。この思想が Defrago のすべての設計判断の根幹にあります。

## 3つの設計原則

| 原則 | 説明 |
|------|------|
| **Zero-Friction Capture** | 書き出す行為のハードルを限りなくゼロに。思いついた瞬間に脳から外部化する |
| **One Screen, One Decision** | 各画面で求める判断は常に1つだけ。迷わない、考えさせない |
| **Think Less, Do More** | アプリが判断フローを導く。管理に頭を使わず、行動に集中する |

## 高速 PDCA — 4つのフェーズ

頭の断片化を解放し、脳のリソースを集中すべきタスクに向けるための PDCA サイクルです。

```
洗い出し → 分類 → 着手 → 見直し → (循環)
```

| フェーズ | できること |
|---------|-----------|
| **洗い出し** | 気になることをすべて Inbox に書き出す。ゼロフリクションで脳を空にする |
| **分類** | ウィザードが Yes/No で導く。タスク・プロジェクト・即実行などに自動分類 |
| **着手** | 今やるべきタスクをフィルタリング。ステータス変更はワンタップ |
| **見直し** | 完了タスクの振り返りとプロジェクトの再計画。サイクルを回し続ける |

### 重要度 × 緊急度マトリクス

| 象限 | 重要度 | 緊急度 | アクション |
|------|--------|--------|-----------|
| **Q1** 必須の領域 | 高 | 高 | 今すぐ実行 |
| **Q2** 効果性の領域 | 高 | 低 | 計画を立てる |
| **Q3** 錯覚の領域 | 低 | 高 | 委任を検討 |
| **Q4** 浪費の領域 | 低 | 低 | 後回し/削除 |

## アーキテクチャ

```
src/defrago/
├── models.py              # データモデル（StrEnum + dataclass）
├── repository_protocol.py # リポジトリProtocol
├── logic/                 # ビジネスロジック（Web非依存）
│   ├── collection.py      # 洗い出し
│   ├── clarification.py   # 分類
│   ├── execution.py       # 着手
│   └── review.py          # 見直し
└── web/                   # FastAPI Webアプリケーション
    ├── app.py             # アプリファクトリ
    ├── routers/           # 各フェーズのルーター
    ├── templates/         # Jinja2 + HTMXパーシャル
    └── static/            # CSS + HTMX
```

### 設計方針

- **ロジック層分離**: リポジトリ Protocol を介し、ビジネスロジックを Web 層から完全分離
- **3層アーキテクチャ**: Model → Logic → Web
- **HTMX**: ページ遷移なしのインタラクティブ UI。SPA の体験をサーバーサイドで実現
- **SQLite 永続化**: 軽量でゼロ設定
