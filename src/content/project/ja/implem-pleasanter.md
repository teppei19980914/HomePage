## プロジェクト概要

大手メーカーの従来の手作業中心の管理業務を自社製品 Pleasanter に置き換え、業務効率化と工数の有効活用を図るプロジェクト。

**PJ規模**: 7名

## 担当業務

- サーバ側（C#）・クライアント側（HTML/CSS/JavaScript）の設計・開発
- 企画工程から開発工程、運用保守工程まで一貫して担当
- お客様折衝および外部ベンダー管理
- API 呼び出し方法の選定（セキュリティリスクを考慮し、拡張 SQL ではなく外部 API 呼び出しを採用）
- アプリケーションエラー（PostgreSQL SEQUENCE 起因）の原因究明と運用手順の改善

## 成果・実績

| 実績 | 内容 |
|---|---|
| 契約額拡大 +33% | プロジェクト契約額を 150万円/1人月 から 200万円/約4人月 に拡大（規模4倍） |
| 信頼獲得 | ステークホルダーとの合意形成・信頼構築により、お客様からの継続的な信頼を獲得 |
| 障害の早期解決 | PostgreSQL SEQUENCE の主キー一意制約違反を自学自習の知識で特定し、早期解決を実現 |
